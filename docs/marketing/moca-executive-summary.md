## מהי MOCA?

MOCA היא מערכת אינטליגנציה תחרותית אוטומטית לתעשיית הסלולר הישראלית, המיועדת למנהלי שיווק הזקוקים לתמונה עדכנית ורציפה של פעילות המתחרים. המערכת מחליפה עבודה סיזיפית — גלישה ידנית יומיומית לאתרי מתחרים, איסוף מסכים, עיבוד בגיליונות אקסל — בתהליך אוטומטי מלא: סריקה, זיהוי שינויים, והתראה בזמן אמת.

**במשפט אחד:** *MOCA עושה את עבודת המעקב התחרותי במקומך, כדי שתוכל להתרכז בהחלטות — לא באיסוף נתונים.*

## הבעיה שהיא פותרת

### עבודת הסיזיף של מנהל השיווק
בכל יום עבודה, מנהלי שיווק בחברות סלולר מבצעים את אותו מחזור מייגע:
- פותחים 8–10 לשוניות דפדפן ובודקים כל מתחרה ידנית
- מחפשים שינויים בהשוואה לזיכרון / צילום מסך מאתמול
- מרכזים ממצאים ב-Slack, אימייל, או אקסל
- חוזרים לבדוק שוב אחר-הצהריים — כי מחירים משתנים כל היום
- מפספסים שינויים שקרו בערב, בשישי, או בין שתי בדיקות

**עלות אמיתית:** 2–3 שעות יום-עבודה לאדם. **עלות אחת שנה:** עשרות אלפי שקלים בשכר + פספוסים תחרותיים שלא ניתן לכמת.

## הפתרון הטכנולוגי

### ארכיטקטורה
```
Frontend (React + Vite + Tailwind)  ←→  Flask API (Python)
  דשבורד השוואה, גרפים, פילטרים         REST API, לוגיקת שינויים
         ↕                                      ↕
  Supabase Auth (JWT, RBAC)          SQLite Database
  ניהול משתמשים ורמות גישה          9 טבלאות, ~3,000 נק' נתונים/יום
                                               ↕
                            Playwright Scraper (40+ scrapers)
                            סריקה headless אוטומטית
                                               ↕
                        Notifier: Telegram / WhatsApp / Email / Web Push
                        התראה מיידית לכל שינוי שזוהה
```

### היקף המעקב האוטומטי
| קטגוריה | היקף | מה זה מחליף |
|---------|------|-------------|
| ספקים מקומיים | 8 (Partner, Pelephone, Cellcom, Hot Mobile, 019, XPhone, We-Com, Neptucom) | 8 לשוניות דפדפן ידנית |
| ספקי נדידה | 8 ספקים × ~15 מדינות = ~120 קומבינציות | גיליון אקסל נדידה שאיש לא מעדכן |
| eSIM גלובלי | 14 ספקים × 200+ מדינות = ~1,800 קומבינציות | עבודה שכלל לא נעשית כיום |
| שירותי תוכן | 5 שירותים × 4 ספקים = 20 מנויים | מעקב ידני אקראי |
| **סה"כ עדכונים/יום** | **~3,000 נקודות נתונים** | **שעות עבודה ידנית** |
| **תדירות סריקה** | **פעמיים ביום (10:00, 16:00)** | **בדיקה אחת בוקר (במקרה הטוב)** |

## הישגים טכניים

**Stack מקצועי ומלא:**
- **Backend:** Python / Flask, APScheduler, SQLite, Playwright (סריקה headless)
- **Frontend:** React 18, Vite, Tailwind CSS, Recharts, React Portal
- **Auth:** Supabase Auth עם JWT, RBAC (admin/viewer), httpOnly cookies
- **Scraping:** 40+ scrapers מותאמים + REST API (Orbit), נרמול שמות מדינות
- **Notifications:** Green API (WhatsApp), Telegram Bot, SendGrid, VAPID Web Push
- **AI:** Claude API לצ'אט ייעוץ מובנה
- **Security:** CSP headers, rate limiting, API key auth, XSS protection
- **PWA:** התקנה על מובייל, Service Worker, אייקוני MOCA

**אתגרי הנדסה שנפתרו:**
- זיהוי False-alarms בשינויי מחיר עקב תנודות שערי מט"ח — נרמול ל-original_price
- Popup dismissal לפני צילום מסך של כל באנר ספק
- ריבוי ספקי eSIM עם מבנה נתונים שונה — מנגנון `getPlanCoverage()` אוניברסלי
- ממשק RTL מלא ב-React ללא ביטול יתרונות Flexbox
